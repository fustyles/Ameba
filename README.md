System Overview
------------------------------------------------------------

This project implements an AI-powered IoT agent running on
the AMB82-mini (Realtek Ameba Pro2 / HUB 8735 Ultra).

It integrates:
- Telegram Bot (HTTPS long polling)
- Google Gemini API (generateContent + vision + search)
- Prompt-based tool calling system (JSON tool_call)
- Camera vision processing
- GPIO / PWM hardware control
- SD card persistent memory
- FreeRTOS background task scheduling

The system behaves as a hybrid AI agent:
"Chat + Tools + Hardware + Vision + Memory"

------------------------------------------------------------
Core Architecture
------------------------------------------------------------

Telegram User<br>
      ↓<br>
getUpdates Polling Task (FreeRTOS)<br>
      ↓<br>
Message Router (Telegram / Direct Command / Gemini)<br>
      ↓<br>
Gemini API (generateContent / search / vision)<br>
      ↓<br>
JSON tool_call Response Parsing (ArduinoJson)<br>
      ↓<br>
Tool Dispatcher (useTools)<br>
      ↓<br>
Hardware / Camera / Memory / Telegram Output

NOTE:
This is a prompt-driven tool routing system.
All tool decisions are generated by LLM output,
and validated on-device via JSON parsing.

------------------------------------------------------------
Main Capabilities
------------------------------------------------------------

- Telegram chatbot via HTTPS long polling (getUpdates)
- Natural language AI assistant (Gemini 3 Flash)
- Tool-based execution via structured JSON output
- Google Search grounding (Gemini tools API)
- Camera capture + Telegram image upload (sendPhoto)
- Vision analysis (image + prompt via Gemini multimodal)
- GPIO digital output control (ON/OFF)
- PWM analog output control (0–255 brightness)

- Automatic WiFi reconnection handling
- FreeRTOS multitasking for continuous polling

------------------------------------------------------------
Supported Telegram Commands
------------------------------------------------------------

/help     Show available commands
/on       Turn ON GPIO device
/off      Turn OFF GPIO device
/pwm      Set PWM brightness (0–255)
/still    Capture and send image to Telegram
/vision   Analyze image using Gemini Vision
/search   Web-grounded AI search
/memory   Show system memory status
/reset    Reset conversation memory

------------------------------------------------------------
Tool Calling Mechanism
------------------------------------------------------------

Gemini is instructed via system prompt to output ONLY
strict JSON tool_call structures when an action is required.

Key design principle:
- No native function calling API is used
- All routing is prompt-based
- Firmware enforces safety via JSON validation

------------------------------------------------------------
Tool Format Specification
------------------------------------------------------------

1. GPIO Digital Output
{
  "type": "tool_call",
  "method": "/on" | "/off",
  "params": {
    "pin": "<GPIO number>",
    "pinmode": "digitalwrite",
    "value": 1 | 0
  }
}

2. PWM Output Control
{
  "type": "tool_call",
  "method": "/pwm",
  "params": {
    "pin": "<GPIO number>",
    "pinmode": "analogwrite",
    "value": 0-255
  }
}

3. Camera Capture
{
  "type": "tool_call",
  "method": "/still",
  "params": {}
}

4. Vision Analysis (Camera + Gemini)
{
  "type": "tool_call",
  "method": "/vision",
  "params": {
    "query": "<user instruction>"
  }
}

5. Web Search (Gemini grounding tool)
{
  "type": "tool_call",
  "method": "/search",
  "params": {
    "query": "<search query>"
  }
}

6. System Tools
{
  "type": "tool_call",
  "method": "/memory" | "/reset",
  "params": {}
}

7. Default Chat Response
{
  "type": "tool_call",
  "method": "/chat",
  "params": {
    "reply": "<natural language response>"
  }
}

------------------------------------------------------------
Execution Pipeline (Firmware Runtime Flow)
------------------------------------------------------------

1. Telegram message received via getUpdates (polling task)
2. Message is classified:
   - Direct command (/on /off /pwm /vision ...)
   - Natural language input → Gemini API
3. Gemini returns tool_call JSON response
4. Arduino parses JSON using ArduinoJson
5. useTools() dispatches execution:
   - GPIO control (digitalWrite / analogWrite)
   - Camera capture (VideoStream API)
   - Vision analysis (Gemini multimodal API)
   - Web search (Gemini grounding tool)
   - Memory operations (RAM + SD storage)
   - Telegram response (sendMessage / sendPhoto)
6. Result is returned to Telegram user

------------------------------------------------------------
Implementation Characteristics
------------------------------------------------------------

- Fully event-driven IoT AI agent
- No webhook dependency (pure polling model)
- Stateless network requests (HTTP POST per call)
- Hybrid AI + embedded system architecture
- Tool execution depends on strict JSON compliance
- Camera frames encoded as Base64 for Gemini Vision

------------------------------------------------------------
System Limitations
------------------------------------------------------------

- RAM growth depends on conversation length
- Vision processing is memory intensive
- No vector database / semantic long-term memory
- Continuous polling increases power usage
- TLS + JSON parsing increases CPU load
- Model output format sensitivity is critical
