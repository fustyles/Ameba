System Overview
------------------------------------------------------------

This project implements an embedded AI-powered IoT agent
running on:

- AMB82-mini
- HUB 8735 Ultra
(Realtek Ameba Pro2 platform)

It integrates:

- Telegram Bot API (HTTPS long polling)
- Google Gemini API (generateContent)
- Gemini multimodal vision reasoning
- Gemini grounded web search
- Prompt-driven JSON tool routing
- Camera capture + image transmission
- GPIO digital / PWM output control
- Digital / analog input sensing
- Persistent SD card memory storage
- FreeRTOS background task scheduling

The system behaves as a hybrid autonomous AI agent:

"Conversation + Reasoning + Tools + Vision + Memory + Hardware"

------------------------------------------------------------
Core Runtime Architecture
------------------------------------------------------------

Telegram User
      ↓
Telegram Polling Task (FreeRTOS)
      ↓
Message Router
(Command / Chat / Gemini Dispatch)
      ↓
Gemini Reasoning Engine
(Chat / Search / Vision / Workflow Planning)
      ↓
Structured JSON tool_call Output
      ↓
ArduinoJson Validation
      ↓
Tool Dispatcher (useTools)
      ↓
Hardware / Vision / Search / Memory Execution
      ↓
Natural Language Feedback to User

NOTE:

This is a prompt-driven orchestration system.

Tool routing decisions are generated by Gemini through
strict JSON output and validated locally before execution.

No native Gemini function-calling API is used.

Tool Calling Mechanism
------------------------------------------------------------

Gemini is instructed through system prompt constraints
to emit ONLY valid JSON tool_call structures whenever
tool execution is required.

Key design principles:

- Prompt-based function routing
- Strict schema enforcement
- Local JSON validation
- Explicit execution sequencing
- No simulated hardware execution

------------------------------------------------------------
Tool Format Specification
------------------------------------------------------------

1. Digital Output

{
  "type": "tool_call",
  "method": "/on" | "/off",
  "params": {
    "pin": "<GPIO>",
    "pinmode": "digitalwrite",
    "value": 1 | 0
  }
}

------------------------------------------------------------

2. PWM Output

{
  "type": "tool_call",
  "method": "/pwm",
  "params": {
    "pin": "<GPIO>",
    "pinmode": "analogwrite",
    "value": 0-255
  }
}

------------------------------------------------------------

3. Digital Input

{
  "type": "tool_call",
  "method": "/digitalread",
  "params": {
    "pin": "<GPIO>",
    "pinmode": "digitalread"
  }
}

------------------------------------------------------------

4. Analog Input

{
  "type": "tool_call",
  "method": "/analogread",
  "params": {
    "pin": "<GPIO>",
    "pinmode": "analogread"
  }
}

------------------------------------------------------------

5. Camera Capture

{
  "type": "tool_call",
  "method": "/still",
  "params": {}
}

------------------------------------------------------------

6. Vision Analysis

{
  "type": "tool_call",
  "method": "/vision",
  "params": {
    "query": "<analysis request>"
  }
}

Capture occurs before multimodal reasoning.

------------------------------------------------------------

7. Grounded Search

{
  "type": "tool_call",
  "method": "/search",
  "params": {
    "query": "<search query>"
  }
}

------------------------------------------------------------

8. Memory Control

{
  "type": "tool_call",
  "method": "/memory" | "/reset",
  "params": {}
}

------------------------------------------------------------

9. Conversational Reply

{
  "type": "tool_call",
  "method": "/chat",
  "params": {
    "reply": "<natural language reply>"
  }
}

Execution Flow
------------------------------------------------------------

1. Telegram receives message
2. Router classifies request:

   - Direct tool command
   - Natural language reasoning request

3. Gemini processes context
4. Gemini emits tool_call JSON
5. ArduinoJson validates structure
6. useTools() executes action:

   - GPIO control
   - Sensor read
   - Camera capture
   - Vision analysis
   - Grounded search
   - Memory operation
   - Telegram output

7. Execution result is injected back into memory
8. Gemini generates natural-language follow-up
9. Final response is sent to user

------------------------------------------------------------
Sequential Task Enforcement
------------------------------------------------------------

For multi-step workflows:

1. Gather external information
2. Analyze completion condition
3. Ask for missing parameters
4. Request user confirmation
5. Execute hardware action
6. Report final state

The model is forbidden from skipping steps
or fabricating completed hardware actions.

In summary, this project connects embedded hardware, cloud multimodal AI, and an instant messaging platform into a truly closed-loop, autonomous agent. Possessing immediate deployment value in fields like smart homes, IoT monitoring, and edge AI computing, this is a rare, high-depth, and fully realized AIoT smart agent reference example within the open-source community.
